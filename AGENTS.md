# Agent Instructions for Lethe

## Project Overview
Lethe is a self-hosted bookmarking app. Public JSON/Markdown data on GitHub, GitHub Actions as the backend, Astro for the static site, Chrome extension for capture.

## Architecture
- **Monorepo** — pnpm workspaces: `site/`, `extension/`, `scripts/`
- **Data layer** — `data/bookmarks.json` (index) + `data/content/{id}.md` (articles). JSON is the source of truth, edited by the extension. Markdown is generated by CI. IDs are ULIDs (26-char Crockford Base32).
- **Site** — Astro 5 + Tailwind CSS + Pagefind search + Zod-validated Content Collections. Data loaded via `site/src/content.config.ts` (Zod schema) and `site/src/lib/bookmarks.ts` (cached fs reads). Base path `/lethe/` for GitHub Pages subpath deployment.
- **Extension** — Manifest V3, Preact + CRXJS + Vite, Dexie.js for offline queue. Uses GitHub Contents API (GET/PUT with SHA) to sync `bookmarks.json`. UTF-8 safe base64 encoding.
- **Backend** — `scripts/process-bookmarks.ts` runs in GitHub Actions. Uses `@mozilla/readability` + `linkedom` for extraction, `sharp` for WebP cover images, `turndown` for HTML-to-Markdown.

## Key Commands
```bash
pnpm install              # install all workspace deps
pnpm dev                  # start Astro dev server (localhost:4321)
pnpm process:bookmarks    # extract content for unfetched entries
pnpm --filter site build  # build static site to site/dist/
pnpm --filter extension build  # build extension to extension/dist/
```

## Data Flow
1. Extension captures URL/title/tags/note → pushes to `data/bookmarks.json` on GitHub
2. GitHub push triggers `.github/workflows/build.yml`
3. Workflow runs `process-bookmarks.ts` → fetches HTML → Readability → Markdown files + WebP covers
4. Workflow commits generated files, builds Astro site, runs Pagefind, deploys to GitHub Pages

## File Conventions
- `data/bookmarks.json` — canonical index. `contentFetched` flag: `false` = needs processing, `true` = done. IDs are ULIDs.
- `data/content/{id}.md` — Markdown with YAML frontmatter (id, url, title). Generated by CI.
- `site/public/assets/covers/{id}.webp` — cached cover images. Generated by CI.
- `site/src/content.config.ts` — Zod schema for bookmark validation (Astro Content Collections)
- `site/src/lib/bookmarks.ts` — cached data loader, reads from `data/` via `process.cwd()` relative paths
- `extension/src/manifest.json` — CRXJS reads this via vite.config.ts
- Extension HTML files (`popup.html`, `options.html`) live at `extension/` root, not `src/`
- `extension/src/lib/ulid.ts` — ULID generator for bookmark IDs

## Tech Constraints
- Node 22, pnpm 9
- Astro reads `data/` via `process.cwd()` + relative paths — works because site runs from `site/` directory
- Site uses `base: '/lethe/'` for GitHub Pages subpath — all internal links use `import.meta.env.BASE_URL`
- Pagefind CSS/JS only loaded in production (`import.meta.env.PROD`) — no 404s in dev mode
- Pagefind JS loaded with `type="module"` (uses `import.meta`)
- Pagefind index is generated at build time via CLI (`pagefind --site dist`), not integrated into Vite dev
- Extension uses `chrome.storage.local` for PAT, config, and default tags — never sent anywhere except GitHub API
- Extension popup sends `SYNC_NOW` message to background service worker after saving
- Extension HTML has NO inline scripts (Manifest V3 CSP blocks them) — dark mode via CSS `prefers-color-scheme`
- Extension popup/options use plain CSS (not Tailwind) — small UI, no utility framework needed
- `chrome.alarms` only used in `background.ts` (service worker context) — not available in popup

## Key Patterns
- **Image fallbacks**: BookmarkCard shows gradient + initial letter when no cover or on image load error
- **Offline sync**: Dexie.js `pendingSync` table with pending/failed/syncing states, 5-minute alarm retry
- **Search**: Pagefind loads at runtime via `type="module"` script; search bar uses DOM API (no innerHTML) for XSS safety
- **Content extraction**: process-bookmarks.ts uses `fetchWithTimeout` (15s), graceful per-bookmark error handling
- **Cards**: Open in new tab (`target="_blank"`)
