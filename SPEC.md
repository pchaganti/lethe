# Technical Specification: Lethe

## 1. Project Overview

**Lethe** is a modern, fast, and self-hosted bookmarking application. It serves as a successor to the `osmosmemo` concept, utilizing current web technologies while remaining 100% free, entirely serverless, and contained within the GitHub ecosystem.

The application operates on a public GitHub repository model. The source data (bookmarks) is publicly readable, hosted on GitHub Pages, and all backend processing (content extraction, image caching, site building) is handled by GitHub Actions.

### Core Principles
- **100% Free Tier:** No external cloud services, no databases, no hosting fees.
- **Git-Native:** Plain-text JSON and Markdown files versioned in Git.
- **Automated Backend:** GitHub Actions fetches article content and caches images on a schedule, keeping the browser extension lightweight and fast.
- **Modern Stack:** TypeScript, Vite, Manifest V3, Astro, and Pagefind.

---

## 2. Architecture & Data Flow

The system consists of three main components: a Browser Extension, a GitHub Repository (acting as the database and CI/CD runner), and a Static Site hosted on GitHub Pages.

### Data Flow
1. **Capture:** The user clicks the browser extension. The extension captures the URL, page title, user tags, and user notes.
2. **Sync:** The extension saves the entry to local IndexedDB (for offline queuing), then fetches the current `bookmarks.json` from GitHub, merges the new entry, and pushes the updated JSON back to GitHub via the Contents API.
3. **Process:** The Git push triggers a GitHub Actions workflow. The workflow identifies new entries missing extracted content. It fetches the article HTML, runs Mozilla's Readability algorithm to extract the clean text, downloads and compresses cover images, and commits these generated files back to the repository.
4. **Build & Deploy:** The workflow then builds the Astro static site, generates the Pagefind search index, and deploys the output to GitHub Pages.

## 3. Technology Stack

| Layer | Technology | Rationale |
|---|---|---|
| **Extension** | TypeScript, Vite, CRXJS | Modern bundling, Manifest V3 support, Hot Module Replacement (HMR). |
| **Extension UI** | Preact, Tailwind CSS | Minimal footprint (~3kb), fast rendering for the popup. |
| **Local Storage** | Dexie.js (IndexedDB) | Offline-first capabilities and sync queue management. |
| **Data Format** | JSON (Metadata) + Markdown (Content) | Diff-friendly, portable, human-readable. |
| **Backend / CI** | GitHub Actions | Free compute for public repos. Handles HTTP fetching, image processing, and deployment. |
| **Content Extraction** | `@mozilla/readability` + `linkedom` | Extracts clean article text without a browser environment. |
| **Image Processing** | `sharp` | Fast WebP conversion and resizing for cached cover images. |
| **Static Site Gen** | Astro 5 | Zero-JS by default, fast builds, excellent content collection APIs. |
| **Search** | Pagefind | Static-site native search. Indexes at build time, ~10kb client runtime. |
| **Hosting** | GitHub Pages | Free, native to GitHub, reliable static hosting. |

## 4. Data Models

The repository uses a hybrid data model: a single JSON file for the index, and individual Markdown files for article content.

### 4.1 `data/bookmarks.json`
This is the canonical index. The browser extension only ever reads and modifies this file.

```jsonc
{
  "version": 2,
  "items": [
    {
      "id": "01JGZ4X2...",            // ULID (Universally Unique Lexicographically Sortable ID)
      "url": "https://example.com/article",
      "title": "Article Title",       // Captured by extension
      "tags": ["ai", "essays"],       // Added by user in extension
      "note": "My personal note",     // Added by user in extension
      "createdAt": "2025-01-15T10:30:00Z",
      "contentFetched": false,        // Set to false by extension, true by GitHub Actions
      "coverImage": null              // Populated by GitHub Actions (e.g., /assets/covers/01JGZ4X2.webp)
    }
  ]
}
```

### 4.2 `data/content/{id}.md`
Generated by GitHub Actions for entries where `contentFetched` was false.

```markdown
---
id: 01JGZ4X2...
url: https://example.com/article
title: Article Title
---

# Extracted Article Title

The clean, readable HTML converted to Markdown by the CI pipeline...
```

## 5. Browser Extension

The extension is strictly a metadata capture and sync tool. It does not fetch article HTML or download images.

### 5.1 Architecture
- **`background.ts` (Service Worker):** Manages the sync queue, handles GitHub API authentication, and performs REST API calls to update `bookmarks.json`.
- **`popup.html` (Preact):** Quick capture UI. Shows the current page title, allows adding tags and notes, and displays sync status.
- **`options.html` (Preact):** Configuration page for the GitHub Personal Access Token (PAT), repository owner/name, and default tags.

### 5.2 Capture & Sync Flow
1. User activates the extension on a webpage.
2. The popup queries the active tab for `title` and `url`.
3. User enters tags/notes and clicks "Save".
4. The entry is saved to a local Dexie database table (`pendingSync`).
5. The service worker wakes up and attempts to sync:
   a. Fetches current `bookmarks.json` and its `sha` from GitHub.
   b. Appends the new entry.
   c. `PUT`s the updated JSON back to GitHub, providing the `sha` to prevent conflicts.
6. If successful, the entry is removed from `pendingSync`. If offline, it retries on a timer.

### 5.3 Authentication
- The user generates a fine-grained GitHub PAT with `Contents: Read/Write` permissions scoped strictly to the Lethe repository.
- The PAT is stored in `chrome.storage.local`.

## 6. GitHub Actions Pipeline (The "Backend")

This is the core of the free-tier automation. Triggered by pushes to `data/bookmarks.json`, it acts as a backend worker.

### 6.1 Processing Script (`scripts/process-bookmarks.ts`)
Runs in Node 22 during the CI workflow.
1. Reads `data/bookmarks.json`.
2. Filters for items where `contentFetched === false`.
3. For each new item:
   - Fetches the HTML via `fetch()`.
   - Parses DOM using `linkedom`.
   - Extracts Open Graph image URLs and favicon URLs.
   - Downloads images, converts them to `.webp` using `sharp`, and saves to `site/public/assets/`.
   - Runs `@mozilla/readability` on the DOM to extract article text.
   - Converts extracted HTML to Markdown and saves to `data/content/{id}.md`.
   - Updates the JSON item: sets `contentFetched = true` and `coverImage = "/assets/covers/{id}.webp"`.
4. Saves the updated `bookmarks.json`.

### 6.2 Workflow Steps
1. Checkout code.
2. Setup Node 22 and pnpm.
3. Run `process-bookmarks.ts`.
4. Git commit and push the newly generated `.md` and `.webp` files, and the updated `bookmarks.json`.
5. Build the Astro site (`astro build`).
6. Run Pagefind indexing (`pagefind --site site/dist`).
7. Upload artifact and deploy to GitHub Pages.

## 7. Static Site (Astro + Pagefind)

### 7.1 Features
- **Dashboard:** A responsive grid of bookmark cards. Cards display the cached cover image (no broken images), title, domain, and tags.
- **Reader Mode:** `/reader/{id}` routes that render the extracted Markdown with beautiful, distraction-free typography.
- **Full-Text Search:** Pagefind indexes all Markdown files, tags, and notes. A search bar (Cmd+K) provides instant, typo-tolerant results entirely client-side.
- **Taxonomy:** Auto-generated pages for `/tags/[tag]` and `/domains/[domain]`.
- **RSS Feed:** Auto-generated `/rss.xml` for integration into RSS readers.
- **Dark Mode:** System-aware theme switching.

### 7.2 Astro Content Collections
The site reads `data/content/*.md` into an Astro Content Collection using a Zod schema, ensuring type safety across the site's components.

## 9. GitHub Actions Workflow Definition

File: `.github/workflows/build.yml`

```yaml
name: Build and Deploy Lethe
on:
  push:
    paths:
      - 'data/bookmarks.json'
  workflow_dispatch:

permissions:
  contents: write   # Required to commit fetched content/images back to repo
  pages: write      # Required to deploy to Pages
  id-token: write

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 22

      - name: Setup pnpm
        uses: pnpm/action-setup@v4
        with:
          version: 9

      - name: Install dependencies
        run: pnpm install --frozen-lockfile

      - name: Process new bookmarks (Fetch content & images)
        run: pnpm run process:bookmarks

      - name: Commit generated content
        run: |
          git config --global user.name "github-actions[bot]"
          git config --global user.email "github-actions[bot]@users.noreply.github.com"
          git add data/ site/public/assets/
          git commit -m "chore: auto-fetch content and images" || echo "No changes to commit"
          git push

      - name: Build Astro Site
        run: pnpm --filter site run build

      - name: Build Pagefind Index
        run: pnpm --filter site run pagefind

      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: site/dist

  deploy:
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: github-pages
    steps:
      - name: Deploy to GitHub Pages
        uses: actions/deploy-pages@v4
```
